## 二、博客数据库字段详细配置

### 字段1: Title（标题）⭐ 必需

**中文名称**: 标题
**英文名称**: Title
**类型**: 标题（Title）
**是否必需**: ✅ 必需（数据库默认字段）

#### 配置说明：
- 这是数据库的默认字段，无法删除
- 用于存储博客文章的标题
- 同步脚本会读取这个字段作为文章标题

#### 配置步骤：
1. 创建数据库时自动生成
2. 可以点击字段名重命名为"标题"或"文章标题"

#### 示例：
```
我的第一篇博客文章
如何使用Notion写博客
Next.js入门指南
```

---

### 字段2: Status（状态）⭐ 必需

**中文名称**: 状态
**英文名称**: Status
**类型**: 单选（Select）或 状态（Status）
**是否必需**: ✅ 必需

#### 配置说明：
- 用于控制文章是否发布到博客
- **只有Status为"Published"的文章才会被同步**
- 可以使用"单选"或"状态"类型

#### 配置步骤：

**方式一：使用"单选"类型（推荐）**

1. 点击数据库右上角的 **"+"** 按钮
2. 在弹出的菜单中选择 **"单选"** （或 **Select**）
3. 字段名称输入：`Status` 或 `状态`
4. 添加以下选项：

| 选项名称 | 颜色 | 说明 |
|---------|------|------|
| Published | 绿色 🟢 | 已发布（会同步到博客） |
| Draft | 黄色 🟡 | 草稿（不会同步） |
| Archived | 灰色 ⚪ | 已归档（不会同步） |

**方式二：使用"状态"类型**

1. 选择 **"状态"** 类型
2. 系统会提供默认的状态选项
3. 确保包含 **"Published"** 选项（名称必须完全匹配，大小写敏感）

#### ⚠️ 重要提示：
```
❌ 错误示例：
- "已发布" （中文不行）
- "published" （小写不行）
- "PUBLISHED" （大写不行）

✅ 正确示例：
- "Published" （首字母大写，其他小写）
```

### 字段3: PublishDate（发布日期）⭐ 必需

**中文名称**: 发布日期
**英文名称**: PublishDate
**类型**: 日期（Date）
**是否必需**: ✅ 必需

#### 配置说明：
- 记录文章的发布日期
- 用于博客文章的时间排序
- 会显示在文章页面上

#### 配置步骤：

1. 点击 **"+"** → 选择 **"日期"** （或 **Date**）
2. 字段名称输入：`PublishDate` 或 `发布日期`
3. 点击字段设置，配置以下选项：

**日期格式选项：**
```
┌─────────────────────────────────┐
│ 日期格式                         │
├─────────────────────────────────┤
│ ○ 友好格式: 2024年1月28日        │
│ ● ISO 格式: 2024-01-28          │← 推荐
│ ○ 美国格式: 01/28/2024          │
│ ○ 欧洲格式: 28/01/2024          │
│ ○ 完整格式: 2024年1月28日 星期日 │
├─────────────────────────────────┤
│ 包含时间                         │
│ □ 包含结束日期                   │
│ □ 包含时区                       │
└─────────────────────────────────┘
```

#### 推荐配置：
- ✅ 日期格式：ISO 格式（2024-01-28）
- ❌ 不要勾选"包含时间"
- ❌ 不要勾选"包含结束日期"
- ❌ 不要勾选"包含时区"

#### 示例：
```
2024-01-28
2024-02-15
2023-12-20
```

### 字段6: Slug（URL路径）⚪ 可选

**中文名称**: URL路径 或 别名
**英文名称**: Slug
**类型**: 文本（Text）
**是否必需**: ⚪ 可选（留空会自动生成）

#### 配置说明：
- 自定义文章的URL路径
- 如果留空，系统会根据标题自动生成

## 四、创建数据库的详细步骤

### 步骤1: 创建新页面

1. 在Notion侧边栏点击 **"+ 新建页面"** 或 **"+ New page"**
2. 或在任意页面中输入 `/database` 并选择"表格"

### 步骤2: 选择数据库类型

选择 **"表格 - 内嵌"** 或 **"Table - Inline"**

### 步骤3: 命名数据库

```
推荐名称：
- 博客文章库
- Blog Posts
- 我的博客
- 文章管理
```

### 步骤4: 添加字段

按照上面的配置说明，依次添加以下字段：

```
1. Title (默认存在) → 保持不变
2. Status → 单选 → 添加 Published, Draft, Archived
3. PublishDate → 日期 → ISO格式
4. Tags → 多选 → 添加常用标签
5. Summary → 文本
6. Slug → 文本（可选）
7. Author → 文本（可选）
8. CoverImage → 文件和媒体（可选）
```

### 步骤5: 调整字段顺序（可选）

拖动字段标题可以调整显示顺序，推荐顺序：
```
Title → Status → PublishDate → Tags → Summary → 其他
```

## 六、常见问题

### Q1: 字段名称必须是英文吗？

**答**：字段名称可以是中文或英文，但为了兼容性，建议：
- **字段名称（显示）**：可以用中文（如"状态"）
- **字段ID（内部）**：使用英文（Status）

实际上，Notion会自动处理，你可以直接用中文。但如果要确保脚本正确识别，建议使用英文字段名。

### Q2: Status的"Published"能用中文"已发布"吗？

**答**：❌ 不能！必须严格使用 `Published`（首字母大写）。

原因：同步脚本中的查询代码是这样写的：
```javascript
filter: {
  property: 'Status',
  select: {
    equals: 'Published',  // 必须完全匹配
  },
}
```

### Q3: 我可以添加其他自定义字段吗？

**答**：✅ 可以！你可以添加任何其他字段，如：
- 阅读时长（数字）
- 难度等级（单选）
- 是否置顶（复选框）
- 相关链接（URL）

这些额外字段不会影响同步，只是不会被导出到博客。

### Q4: 日期格式必须是ISO格式吗？

**答**：不必须，但强烈推荐。ISO格式（2024-01-28）是国际标准，便于程序处理。

### Q5: 标签的颜色重要吗？

**答**：不重要。颜色只是在Notion中用于视觉区分，不会影响博客的显示。
